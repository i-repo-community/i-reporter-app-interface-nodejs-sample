§ 07 — TODAY'S STACK
今回の構成（技術スタック）

今回は、AWS Lambda のご紹介

CDK は使いません。コンソールなど最短の手順で関数を立ち上げ、Lambda の簡単さ（呼ばれたときだけ動き、JSON を返せばよい）をまず体感してもらいます。

LAYER 01
AWS Lambda

i-Reporter からのリクエストを受けて、JSON を返す関数。

LAYER 02
API Gateway

HTTPS の URL を用意し、Lambda に届ける窓口。まずはここまでつなげば、i-Reporter から叩けます。

Lambda でやること

リクエストを受けて決まった形の JSON を返す。

index.mjs（ハンドラー一式）
/**
 * i-Reporter App Interface 向けの最小レスポンス例。
 * シート1・クラスター1 に「ハロー」を書き込む JSON を返します。
 *
 * Lambda コンソールの既定テンプレに近い export const handler 形式。
 * API Gateway（Lambda プロキシ統合）から呼ばれる想定。
 */
export const handler = async (event) => {
  // TODO: 必要なら event をログ
  // console.log(JSON.stringify(event, null, 2));

  const appInterfaceResponse = {
    result: {
      code: 0,
      description: "OK",
    },
    apply: [
      {
        sheet: 1,
        cluster: 1,
        type: "string",
        value: "ハロー",
      },
    ],
  };

  const response = {
    statusCode: 200,
    headers: {
      "Content-Type": "application/json; charset=utf-8",
      "Access-Control-Allow-Origin": "*",
    },
    body: JSON.stringify(appInterfaceResponse),
  };
  return response;
};

提供する2つの外部連携構築方法
Option 01

ConMas Gateway による連携

Python でアクションファイルを作成し、Gateway サーバーに配置することで外部システムと連携する。弊社フレームワークを使用するため、サポートが充実している。

  • Proシムトップスがサポートできる連携方法
  • ConPython 縛り／Gateway サーバー管理が必要

「i-Reporter App Interface」仕様に従って JSON レスポンスを返す仕組みを実装するだけ。ConMas Gateway を利用せずとも、あらゆるプログラミング言語やフレームワークで連携可能。

  • ProPython 縛りから解放、多様な技術スタックで開発可能

§ 02 — RESPONSE SPEC
レスポンス仕様 / 基本構造

とってもシンプル

i-Reporter が読むのは、result と apply[] の2キーだけ。余計なフィールドも独自拡張も不要で、挙動は読み取りやすい。

01 / 05 — Result

ステータスを返す

result.code は 0 で成功。それ以外の値を返すと、description がエラーメッセージとして表示される。

response.json
{ "result": { "code": 0, "description": "データ取得成功" }, "apply": [ { "item": "factory-name", "sheet": 1, "cluster": 2, "type": "string", "value": "東京工場" }, { "item": "address", "sheet": 1, "cluster": 3, "type": "string", "value": "〒105-0011 東京都港区芝公園…" } ] }

§ 05 — CUSTOM MASTER
カスタムマスター連携

マスター選択を、
API と結びつける。

i-Reporter のマスター選択クラスターと API を連携できる。ConMas Designer が自動で /fields /params /getrecords をエンドポイント末尾に追加する。

対応する型
text
テキストクラスター
date
年月日クラスター
bool
チェッククラスター
numeric
数値クラスター
time
時刻クラスター
Endpoint — /fields

フィールド定義を返す

fields/name は表示名。fields/item は任意のアイテム名。no はレコードと対応付けるための番号。

GET /api/v1/master/fields
{ "result": { "code": 0, "remarks": ["商品フィールド取得成功"] }, "fields": [ { "no": 1, "name": "product_id", "type": "text", "item": "product_id" }, { "no": 2, "name": "product_name", "type": "text", "item": "product_name" }, { "no": 4, "name": "price", "type": "numeric", "item": "price" }, { "no": 6, "name": "is_active", "type": "bool", "item": "is_active" }, { "no": 7, "name": "created_date", "type": "date", "item": "created_date" } ] }
Endpoint — /params

検索パラメータを返す

検索時にクライアント側から送られるパラメータの定義。name と type のみ。

GET /api/v1/master/params
{ "result": { "code": 0, "remarks": ["パラメータ取得成功"] }, "params": [ { "name": "product_id", "type": "string" } ] }
Endpoint — /getrecords  ·  POST

レコードを返す

検索結果のレコード配列を返す。defaultRecord は任意、records が空の場合のデフォルト値。

Warning ⚠   レコード数は 500 程度 が限度。bool は "true"/"false" 文字列、date は "YYYY/MM/DD"。
POST /api/v1/master/getrecords
{ "result": { "code": 0, "remarks": ["レコード取得成功", "1件のレコードを取得"] }, "fields": [ /* /fields と同じ定義 */ ], "defaultRecord": { "fields": [ { "no": 1, "value": "1001" }, { "no": 2, "value": "製品A" }, { "no": 4, "value": "15000" }, { "no": 6, "value": "true" }, { "no": 7, "value": "2024/01/15" } ] }, "records": [ { "fields": [ /* no & value の配列 */ ] } ] }
§ 06 — IMPLEMENTATION NOTES
ご注意事項

Tips

NOTE 01 · REQUEST

POST は multipart/form-data

i-Reporter からは multipart/form-data 形式で送られる。Node.js では multer を使うと便利。

NOTE 02 · REQUEST

GET はクエリで座標を送る

?temperature={1,5} — 1ページ目、クラスター index 5 を意味する。シート番号は省略可。

NOTE 03 · VALUE

クリアは空文字で

クラスターをクリアする場合は ""（空文字列）を使う。null は指定できない。

NOTE 04 · DATE

日付は string で送る

専用型は存在しない。type: "string" で "2025/9/1" のような文字列を送信する。

NOTE 05 · IMAGE

Base64 プレフィックスは不要

data:image/png;base64, は除去して、Base64 文字列のみを value に格納する。

NOTE 06 · SAMPLE

Node.js サンプル on GitHub

i-repo-community / i-reporter-app-interface-nodejs-sample に Node.js サンプルと i-Reporter 帳票定義を公開中。
